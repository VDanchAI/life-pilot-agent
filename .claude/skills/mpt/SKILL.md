# MPT TOOL v1.2 — Multi-Persona Thinking Protocol

## СУТЬ МЕТОДА

MPT — это **многоагентная дискуссия**, в которой три субагента с изолированным контекстом отстаивают разные роли, а оркестратор ведёт протокол и собирает итог.

Каждый субагент работает в своём контексте и видит только то, что передаёт ему оркестратор. Это ломает главную проблему обычного анализа — sycophancy: агент-защитник физически не может "скоситься" на позицию оппонента до нужного момента.

**Почему субагенты, а не роли внутри одного промпта:**
Один Claude знает, что сейчас "играет" оппонента — и подсознательно смягчает удары. Изолированный субагент просто выполняет свою задачу от начала до конца, не зная об остальных.

## ПРОТОКОЛ ВЫПОЛНЕНИЯ

### Шаг 0 — Подготовка (оркестратор)

Перед запуском чётко сформулировать задачу:
```
"Какое конкретно решение анализируем? [вариант X] vs [вариант Y]?"
```
Выбрать названия ролей под контекст (см. таблицу ниже).

---

### Шаг 1 — Раунд 0: параллельный запуск A и B

Оркестратор запускает двух субагентов **одновременно** (параллельно).

**Промпт для Субагента A:**
```
Ты — [РОЛЬ A, например "Архитектор масштабирования"].
Твоя задача: аргументировать ЗА следующее решение максимально сильно.

РЕШЕНИЕ: {описание}

Правила:
- Никаких оговорок, никаких "но"
- Аргументируй как адвокат в суде — только в защиту
- Конкретика: цифры, факты, прецеденты
- Выяви, почему именно это решение оптимально
```

**Промпт для Субагента B:**
```
Ты — [РОЛЬ B, например "Адвокат простоты"].
Твоя задача: найти ВСЁ что может пойти не так с этим решением.

РЕШЕНИЕ: {описание}

Правила:
- Ищи скрытые допущения: "А мы вообще проверяли, что X правда?"
- Давай конкретные сценарии провала: "Что если через 3 месяца...?"
- Не пытайся быть конструктивным — твоя задача найти слабые места
- Жёстко. Конкретно. Без дипломатии.
```

Оркестратор собирает оба вывода.

---

### Шаг 2 — Раунд 0: запуск арбитра C

Оркестратор передаёт Субагенту C **оба** вывода A и B.

**Промпт для Субагента C:**
```
Ты — нейтральный арбитр. Тебе передали два набора аргументов по одному решению.

АРГУМЕНТЫ ЗА (Субагент A): {вывод A}
АРГУМЕНТЫ ПРОТИВ (Субагент B): {вывод B}

Твоя задача — оценить качество аргументации:
- Где A прав? Какие аргументы A невозможно оспорить?
- Где B прав? Какие риски B реальны?
- Логические ошибки A: что преувеличивает или упускает?
- Логические ошибки B: что преувеличивает или упускает?
- Что ОБА упускают? Есть ли третий вариант который никто не назвал?

Не занимай сторону. Оценивай качество аргументов, а не их количество.
```

---

### Шаг 3 — Раунд 1: параллельная диалектика A и B

Оркестратор передаёт каждому агенту критику арбитра и аргументы противника.

**Промпт для Субагента A (Раунд 1):**
```
Ты — [РОЛЬ A]. В Раунде 0 ты выдвинул аргументы ЗА.

Твои аргументы из Раунда 0: {вывод A}
Что сказал оппонент (Субагент B): {вывод B}
Что отметил нейтральный арбитр: {вывод C}

Теперь:
- Что принимаешь из критики B как справедливое?
- Где B всё равно ошибается — и почему?
- Усиль свою позицию с учётом анализа арбитра.
```

**Промпт для Субагента B (Раунд 1):**
```
Ты — [РОЛЬ B]. В Раунде 0 ты нашёл слабые места решения.

Твои аргументы из Раунда 0: {вывод B}
Что сказал защитник (Субагент A): {вывод A}
Что отметил нейтральный арбитр: {вывод C}

Теперь:
- Какие аргументы A признаёшь сильными?
- Что в твоей позиции подтвердил арбитр?
- Появились ли новые риски в ходе дискуссии?
```

---

### Шаг 4 — Финальный арбитраж C

Оркестратор передаёт C все четыре вывода (A₀, B₀, A₁, B₁).

**Промпт для Субагента C (финальный):**
```
Ты — нейтральный арбитр. Вот полная дискуссия двух раундов:

Раунд 0 — A: {A₀}
Раунд 0 — B: {B₀}
Раунд 1 — A: {A₁}
Раунд 1 — B: {B₁}

Вынеси финальный вердикт:
- Кто убедительнее после двух раундов и почему?
- Какие противоречия остались неразрешёнными?
- Конкретная рекомендация арбитра.
```

---

### Шаг 5 — Финальный синтез (оркестратор)

Оркестратор пишет синтез сам, опираясь на все выводы:

```
───── ФИНАЛЬНЫЙ СИНТЕЗ ─────

РЕШЕНИЕ: [конкретный выбор — не "зависит от контекста"]
ОБОСНОВАНИЕ: [с учётом всех аргументов двух раундов]
ПЛАН ДЕЙСТВИЙ: [конкретные шаги]
УСЛОВИЯ ПЕРЕСМОТРА: [при каких новых данных решение меняется]
ВЫЯВЛЕННЫЕ BIAS: [какие искажения нейтрализованы в ходе дискуссии]
```

## КОГДА ЗАПУСКАТЬ

```
□1 Это необратимое решение? (архитектура, стратегия, стек, дизайн)
□2 Есть только одна позиция без контраргументов?
□3 Кто-то убедительно продвигает конкретное решение?
```

**Триггер:** `□1 ИЛИ (□2 + □3)`

Оркестратор **предлагает**, а не запускает автоматически:
```
"Тут стоит прогнать MPT — [причина]. Запускать?"
```

**Не запускать:** обратимые решения за 5 минут, рутина, синтаксис, стилистика.

## ВЫБОР РОЛЕЙ

| Задача | Роль A (ЗА) | Роль B (ПРОТИВ) |
|--------|-------------|-----------------|
| Архитектура | Инноватор-оптимизатор | Хранитель стабильности |
| Стратегия бизнеса | Growth-oriented | Risk-averse |
| Новый инструмент | Early adopter | Минималист-скептик |
| Выбор технологии | Сторонник X | Сторонник альтернативы Y |
| Планирование | Оптимист по срокам | Пессимист по рискам |
| Ценообразование | Высокая цена = ценность | Цена отсекает рынок |

Роли должны быть **максимально различными**. Называть конкретно: "Архитектор масштабирования" — не просто "Сторонник".

**Роль C — всегда нейтральный арбитр.** Не адаптируется под домен.

---

## АНТИПАТТЕРНЫ

- **Субагент A видит промпт B до своего вывода** — нарушает изоляцию, весь смысл теряется
- **Мягкий B** — "в целом решение верное, но..." → переформулировать задачу B жёстче
- **Арбитр-дипломат** — "истина посередине" → потребовать конкретную рекомендацию
- **Пропуск Раунда 1** — роли не корректируют позиции → реальная работа не происходит
- **Оркестратор пишет синтез без финального арбитража** → теряется 33% качества
